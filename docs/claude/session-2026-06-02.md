## 5. PDF-utskrift (`a3e9869` → `8b45e86`)

Spec: `prompt-pdf-storrelse-zoom-maal.md` (to versjoner) +
`prompt-pdf-en-side-pr-rom.md`.

### v1 (`a3e9869`)
- `_renderRoomToImage`: PNG → JPEG, beskjær til rommets fokusboks (rom fyller
  bildet i stedet for tom kontekst), tak 1400 px, padding 120→150 cm.
- Tvinger på folie-/kabel-/matte-målkjeder i utskrift — men kun for varmetyper
  rommet faktisk har (unngår at rom-målkjeden blåses langt ut). Avvik fra spec
  (som ba om alle tre alltid) — rapportert til Kenneth.
- `exportPDF` bruker JPEG + reell aspekt (ikke fast 0.70).

### v2 (`8b45e86`) — skarphet ble prioritet, ikke filstørrelse
- Render rom-bildet **1:1 i mål-oppløsning** i stedet for å oppskalere det grove
  skjermlerretet: setter midlertidig `canvas.width/height` til ~300 DPI av
  A4-trykkflaten (`outW ≈ 2126 px`, hard tak 2600 px), zoomer så fokusboksen
  mapper nøyaktig på lerretet, fanger hele lerretet til JPEG q=0.9. Gjenoppretter
  canvas-størrelse + view etterpå.
- **Alt om ett rom på ÉN A4-side**: teller spec-rader på forhånd
  (`_specRowCount`), budsjetterer bildehøyden
  (`availImgH = (pageH−bunn) − y − metricsH − specH − gaps`, gulv 55 mm), og
  fjernet `_checkPage` mellom bilde / nøkkeltall / spesifikasjon. Høydebegrenset
  bilde skaleres med beholdt aspekt og sentreres. `_checkPage(5)` i rad-løkken
  beholdt som robusthet for rom med svært mange produkter.
