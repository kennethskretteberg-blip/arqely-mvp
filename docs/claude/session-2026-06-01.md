## Hva ble bygget

Porter den verifiserte `engine2`-motoren fra prototypen
(`docs/claude/kabel-prototype-skra.html`) inn i `romtegner.html`. Mål
(spec §2.7): hjørne-til-hjørne serpentin, lik CC, halvsirkel-U-svinger
som buler UTOVER ("vend utover"), ingen hopp over åpent gulv — også for
skråvegger og hindringer.

### Arkitektur — additiv (ingen regresjon)

Den eksisterende kabelmodellen er **kun aksejustert** (`cable.runs[]` +
`cable.direction` 'v'/'h' + `segments[]`), og kan ikke representere
roterte runs. Løsning: skew-kabler bærer et NYTT felt `cable.pathEls` =
ordnede verdens-elementer:

- `{kind:'run', a, b}`
- `{kind:'uturn', from, to, nrm}` (buler langs `nrm`)
- `{kind:'connector', from, to}` (vegg-hug L rundt hull)

Buer samples ved RENDER-tid (kompakt lagring, skarpt ved alle zoom).
Aksejusterte kabler er helt urørt → ingen regresjon. Skew-kabler er
auto-fill-only (drag-redigering deaktivert i v1).

## Endringer (alt i `romtegner.html`)

### Motor (før `generateCableBoustrophedon`)

- `_sk*`-hjelpere: `_skRotate`, `_skBbox`, `_skClipVertical` (even-odd
  y-intervaller), `_skSubtract`, `_skOverlap`, `_skUF` (union-find),
  `_skPolyArea`.
- `_skGenerateRotated(roomR, holesR, CC, margin)` — run-grid +
  boustrophedon-celler i rotert frame, 1:1 lane-linking.
- `_skSerpCell(cell, ltr, entryLow)` — serpentin én celle → ordnede
  elementer (run/uturn alternerende).
- `_skAdjacent`, `_skBuildPath(model, …)` — greedy walk over
  celle-naboskap, scorer `-openLen*10 + hjørne-bonus`.
- `_skEngineCore(room, holes, CC, margin, angleDeg)` — prøver
  kandidatvinkler (hver kant-retning mod 180 + 0/90), velger beste ved
  `cov - openLen*CC*2`; roterer resultat til verden.
- `generateCableSkew(roomId, productId)` — løser rom/prod/margin, hull =
  `S.hindrings` + forbudte `S.zones`; CC-biseksjon (24 iter) for
  fastlengde-produkter; returnerer additiv kabel-form eller null.

### Renderer

- `drawCables`-guard: `_hasGeom = runs.length>0 || pathEls.length>0`.
- `_drawCableSkewPath(cable)` — runs som linjer, U-svinger som
  halvsirkel-buer (16 segm, buler langs `el.nrm`), connectors som
  stiplet vegg-hug L.
- Label-blokk + label-drag: pathEls bbox-senter-gren.

### Endepunkter / hit-test

- `_cableEndpoints`: returnerer `cable.start`/`cable.end` direkte for
  skew-kabler.
- `hitCable`: per-element `distSeg`-gren for pathEls.
- Gizmo-drag: deaktivert for skew-kabler (`pathEls.length` → ingen drag).

### Innkobling i `autoFillCable`

Skew-forsøk lagt inn **etter** boustrophedon, **før** V6, gated:

```js
const needSkew = (!rectilinear) || hasHindring || hasForbidden;
if (needSkew) {
  const res = generateCableSkew(roomId, productId);
  if (res && res.pathEls?.length && res.coverage >= 70 &&
      (res.openLen||0) < spacing*1.5) {
    return { …, _engine:'skew', pathEls, start, end, angleDeg,
             spacing_cm, cc_mm, totalLength_cm, runs:[], connections:[] };
  }
  // ellers fall gjennom til V6/V5/V4
}
```

`rectilinear` = alle kanter aksejusterte (dx≈0 eller dy≈0). Rettvinklede
rom uten hull når aldri skew-blokken → eksisterende motorer urørt.

## LOCKED-regler respektert

Halvsirkel U-svinger r=CC/2; `sweepMargin = margin + CC/2`; lik
strand-lengde per celle; ingen Y-splits; bue-retning utover. Ingen
endring i eksisterende aksejusterte motorer.

# Cowork-økt (research/design — lå til grunn for porteringen over)

Egen økt i Cowork SAMME dag, FØR Claude Code-porteringen. `romtegner.html`
ble ikke rørt her — målet var å finne og **bevise** algoritmen i kjørbare
prototyper, så porteringen over hadde en verifisert mal.

## Hva ble gjort

1. **Kartla dagens kabelmotor.** `autoFillCable` kjører fem motorer i
   kaskade (Serpentine→V6→V5→V4→length-driven). Global utjevning (LOCKED)
   tvinger kabelen inn i ett rektangulært bånd → L/T-rom får udekket utstikk
   eller slutt midt i rommet. Hjørne-landing var bare en myk score-bonus.

2. **Research** (boustrophedon coverage path planning: Choset & Pignon;
   Bähnemann m.fl. 2019 — fast start/slutt-node).

3. **Prototype 1** (`kabel-prototype.html`) — rettvinklede rom. Bygde global
   CC-rute + per-celle-utjevning + paritets-styrt celleovergang. Kenneth
   fanget en feilversjon som **hoppet** fra topp av høy romdel rett ned til lav
   romdel (skjermbilde). Fikset: kabelen forlater hver celle gjennom den DELTE
   kanten → overgangen blir en vanlig U-sving (`openLen=0` på L).

4. **Prototype 2** (`kabel-prototype-skra.html`, `engine2`) — generell:
   - Skråvegger via rotert sveiperamme (runs parallelle med vegg).
   - Hindringer via boustrophedon-celler (1:1 lane-link, split/merge=ny celle).
   - **Bueretning:** Kenneth fanget at buer bulet feil vei. Fikset til lokal
     regel (vend utover, `y1>y0`→opp) = `wallDir` i `_drawCableUTurns`. Den
     gamle «vekk fra rom-sentrum»-heuristikken bommet ved hindringer.
   - UI: zoom/pan + flyttbare bokser (HUD dekket S-punktet).

5. **Skrev spec** `kabel-hjorne-til-hjorne-spec.md` (§2.7 + prompter §5/§5b),
   med ærlige grenser (fri hindring → slutt ved hindring; trekantspiss ~20cm)
   og presiseringer fra Kenneths review (søk funksjonsnavn ikke linjenr;
   behold aksejustert hurtigsti).
