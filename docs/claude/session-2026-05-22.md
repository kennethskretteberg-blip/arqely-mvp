## Hva ble bygget

Ny tegnemodus `hindring-w2w` i `romtegner.html`. Bruker klikker første
vegg, valgfrie mellompunkter, andre vegg → rommet deles i to halvdeler
→ bruker klikker den halvdelen som skal bli hindring.

Gjenbruker `splitRoomWithPolyline(room, polyline, startWallIdx,
endWallIdx)` (linje ~19868) fra skillevegg-flowen. Forskjellen er at
hindring-w2w ikke endrer rommet — den valgte halvdelen blir bare en
ny hindring innenfor rommet.

## Endringer

Alt i `romtegner.html`, +236 linjer.

### State

`S.drawing.hindringW2W = { startWallIdx, startPt, endWallIdx, endPt,
phase, pendingSplit, hoverSide }` lagt til to steder:

- I `S.drawing`-initdef ved siden av `skillevegg` (~linje 2095).
- I `exitDrawing()` reset (~linje 17898). Eksisterte allerede fra forrige
  økt — bekreftet at den nullstilles ved Esc.

### Mousedown (~linje 22028)

Ny `else if (mode === 'hindring-w2w')`-gren etter skillevegg:

- **Fase 0 → 1**: `snapToRoomWall(wp.x, wp.y, room, threshold)`; hvis
  treff: lagre `startWallIdx`, `startPt`, push punkt, `phase = 1`.
- **Fase 1**: hvis snap til annen vegg → kall `splitRoomWithPolyline`,
  valider via inline `_area`-helper (`>= 100 cm²` per halvdel), lagre
  `pendingSplit = { polyA, polyB, polyline, room }`, `phase = 2`. Hvis
  snap til samme vegg → alert. Ellers angle-snappet waypoint.
- **Fase 2**: `ptInPoly` mot polyA/polyB, push hindring med
  `S.hindringType`, `S.hindringCustomName`, `roomId = room.id`.
  `exitDrawing()`.

### Mousemove (~linje 23227)

Etter skillevegg-fase-2-blokken: ny gren som kjører `ptInPoly` mot
begge polygoner i fase 2, oppdaterer `hoverSide` ('A' | 'B' | null) +
cursor + `markDirty()`. Tom oppdatering noops.

### drawPreview (~linje 16150)

Ny `if (mode === 'hindring-w2w')`-blokk etter skillevegg-preview:

- Highlight start-wall i oransje (fase ≥ 1).
- Fase 0/1: dashed line til cursor + segment-lengde + endepunkt-vegg
  highlightes grønn ved snap.
- Fase 2: begge halvdeler rendres — default grå-stiplet med "Klikk her"
  sentroide-label, hover oransje med "Klikk for hindring". Polylinen
  tegnes på toppen i oransje.

### updHint (~linje 20302)

Tre nye case for fase 0/1/2:
- Steg 1/3: Klikk på en vegg for å starte hindringen
- Steg 2/3: Klikk på motstående vegg ... eller mellompunkt
- Steg 3/3: Trykk på området som skal være hindring

### Entry point

Knapp `⤬ Vegg-til-vegg →` lagt til i hindring-modalen ved siden av
`⬡ Tegn polygon →`. Ny funksjon `confirmHindringW2W()` etter
`confirmHindringPolygon()` — setter `drawMode`, initialiserer state,
lukker modal, setter crosshair-cursor.

### Escape

Allerede dekket av eksisterende generisk `cancelDrawing()` →
`exitDrawing()` siden state-reset er på plass.

## Verifisering

Programmatisk simulering via preview-eval:

```
Fase 0 → klikk bunnvegg (y=0, x=100): phase=1, startWall=0, pts=1
Fase 1 → klikk midt i rom (100, 150): phase=1, pts=2
Fase 1 → klikk topvegg (y=300, x=200): phase=2, pendingSplit satt,
                                        polyA=5 pkt, polyB=5 pkt
Fase 2 → klikk i venstre halvdel (50, 150): hindring opprettet med
                                              5 punkter, drawMode=null
```

Også verifisert at `render()` ikke kaster i fase 2 og at hint-tekst
viser riktig per fase.

## Edge cases dekket i koden

- Klikk samme vegg som start → alert "må gå mellom to forskjellige
  vegger".
- Ugyldig split (areal < 100 cm² eller polygon < 3 pkt) → alert, reset
  til fase 0.
- Klikk utenfor rommet i fase 1 → ignoreres, kun snap-til-vegg eller
  waypoint registreres.
- Klikk utenfor begge halvdeler i fase 2 → ingen action (venter på
  unambiguous click).
